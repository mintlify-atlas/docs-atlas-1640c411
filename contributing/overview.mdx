## Why Contributions Matter

The Nigerian Fintech LLMs.txt project bridges a critical gap between AI-powered development tools and Nigerian fintech platforms. While developers increasingly rely on Large Language Models (LLMs) to accelerate integration work, these AI tools struggle to extract relevant information from large, complex documentation sites.

By contributing `llms.txt` files, you're:
- **Enabling faster development** - AI coding assistants can quickly understand API structures
- **Improving accuracy** - Structured documentation reduces hallucination and incorrect implementations
- **Supporting the ecosystem** - Open-source documentation benefits all Nigerian developers
- **Encouraging adoption** - Companies can use these files as templates for official implementation

<Note>
This project was created to help fintech companies adopt the [`llms.txt` standard](https://llmstxt.org/) for AI-friendly documentation.
</Note>

## What You Can Contribute

We welcome contributions in several forms:

### New Provider Documentation
Add `llms.txt` files for Nigerian fintech platforms not yet in the repository:
- Payment gateways
- Open banking platforms
- Digital lending APIs
- Mobile money services
- KYC/identity verification providers

### Updates to Existing Files
Improve existing documentation with:
- New API endpoints or features
- Corrected information
- Better organization or clarity
- Additional SDKs or integration guides

### Format Improvements
Suggest enhancements to the standard structure itself.

## Contribution Process Overview

Contributing follows a standard open-source workflow:

<Steps>
  <Step title="Research the provider">
    Gather comprehensive documentation about the fintech platform's APIs, SDKs, and integration guides.
  </Step>
  
  <Step title="Create the llms.txt file">
    Follow our [format guide](/contributing/format-guide) to structure the documentation properly.
  </Step>
  
  <Step title="Test your file">
    Verify the file works with AI tools and follows the standard structure.
  </Step>
  
  <Step title="Submit a pull request">
    Follow our [step-by-step guide](/contributing/adding-providers) to submit your contribution.
  </Step>
</Steps>

For detailed instructions, see our [Adding Providers guide](/contributing/adding-providers).

## Code of Conduct

We're committed to maintaining a welcoming, inclusive community:

- **Be respectful** - Treat all contributors with courtesy and professionalism
- **Be constructive** - Provide helpful feedback and suggestions
- **Stay focused** - Keep discussions relevant to improving documentation
- **Assume good intent** - Give contributors the benefit of the doubt
- **Be patient** - Remember that maintainers and contributors volunteer their time

<Tip>
If you're unsure about whether a provider fits the project scope, open an issue on GitHub to discuss before investing time in creating the documentation.
</Tip>

## Quality Standards

All contributions should:
- Accurately represent the provider's official documentation
- Follow the established `llms.txt` format
- Include both documentation overview and API reference sections
- Use clear, consistent language
- Be tested with at least one AI tool

## Getting Help

Need assistance with your contribution?

- **Review existing files** - Check [Mono](https://github.com/sin4ch/nigerian-fintech-llms/blob/main/mono/llms.txt), [Paystack](https://github.com/sin4ch/nigerian-fintech-llms/blob/main/paystack/llms.txt), or [OPay](https://github.com/sin4ch/nigerian-fintech-llms/blob/main/opay/llms.txt) for examples
- **Open an issue** - Ask questions on the [GitHub repository](https://github.com/sin4ch/nigerian-fintech-llms)
- **Check the format guide** - Our [format specification](/contributing/format-guide) covers common questions
